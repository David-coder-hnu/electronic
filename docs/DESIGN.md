## Problem Statement
在Cyclone IV FPGA开发板上实现一套蓝牙无线控制的RGB彩灯系统。核心链路：手机APP → 蓝牙 → FPGA(Verilog) → PWM → RGB LED。课程要求覆盖数字逻辑设计、呼吸灯算法、蓝牙通信、PCB设计四大能力。

## What Makes This Cool
- **全栈硬件闭环**：从手机APP到底层Verilog全自研，没有Arduino、没有软核CPU、没有现成模块。教授看到的是"一个学生用纯硬件逻辑完成了无线灯控"。
- **演示即产品**：拿出手机，打开自研APP，选颜色/切模式 → 灯秒变。这不是实验板在闪，这是个"东西"在work。
- **音乐律动潜力**：MVP打通全链路后，加一个麦克风模块就能升级为音乐灯光秀——这是演示炸场的王牌。

## Constraints
- 必须基于Cyclone IV FPGA，Verilog实现所有控制逻辑，禁止软核CPU
- 蓝牙通信通过PMOD模块实现
- 必须自研手机APP（非串口调试助手）
- 至少支持颜色切换 + 2种以上灯效（流水、呼吸）
- 自制RGB PMOD扩展板（考虑8 IO利用率和多板堆叠可行性）

## Premises
1. 核心价值是"全栈硬件技能"——Verilog+PWM+蓝牙+PCB+APP每个环节都自己做
2. 音乐律动是加分项——先打通蓝牙调色MVP再考虑FFT
3. 课堂演示效果比代码优雅更重要
4. 自制PMOD灯板是必须的，要焊出实物

## Approaches Considered

### Approach A: 串口调灯原型
- 用现成蓝牙串口助手代替自研APP
- 专注Verilog核心逻辑：UART解析 + PWM驱动 + 2种灯效
- 标准PMOD RGB LED板
- 优点: 最快跑通(~3天)，缺点: 演示效果不够"产品感"

### Approach B: 蓝牙调色MVP (已选择)
- 自研手机APP（色轮拾取 + 模式切换 + 场景保存）
- Verilog全硬核：UART解析 → 指令解码 → 效果引擎 → PWM驱动
- 自制RGB PMOD板（8 IO充分利用）
- 优点: 全栈闭环，演示效果好，技能覆盖完整

### Approach C: 音乐律动灯光秀
- 方案B全功能 + 音频ADC采样 + Verilog包络检测
- 多灯板扩展
- 优点: 最酷，缺点: 风险高，时间紧张可能两头不讨好

## Recommended Approach
**方案B：蓝牙调色MVP**。分三阶段推进：

```
Phase 1: 硬件基础层
  Verilog PWM驱动(3路RGB) → 呼吸灯效果 → 流水灯效果
  自制PMOD RGB灯板(PCB设计→打样→焊接)

Phase 2: 通信链路层
  Verilog UART解析器 → 指令协议设计 → 蓝牙PMOD对接
  手机APP v1.0 (色轮 + 模式切换 + 蓝牙连接)

Phase 3: 集成与演示
  全链路联调 → 效果优化 → 演示准备
  加分项(时间允许): 音频输入 → 包络检测 → 音乐联动
```

## PMOD 端口分配 (双PMOD口)
```
PMOD-A (蓝牙 CH9143):
  Pin 1: VCC 3.3V    Pin 7: BLESTA (连接状态输入)
  Pin 2: GND         Pin 8: RXD (FPGA←CH9143 TXD)
  Pin 3: TXD (FPGA→CH9143 RXD)
  Pin 4-6: 预留 (可用于 AT 模式切换等)

PMOD-B (RGB LED板):
  Pin │ Signal  │ Description
  ─────┼─────────┼─────────────────
  IO[0]│LED1_R  │ LED1 Red PWM
  IO[1]│LED1_G  │ LED1 Green PWM
  IO[2]│LED1_B  │ LED1 Blue PWM
  IO[3]│LED2_R  │ LED2 Red PWM
  IO[4]│LED2_G  │ LED2 Green PWM
  IO[5]│LED2_B  │ LED2 Blue PWM
  IO[6]│LED3_R  │ LED3 Red (预留)
  IO[7]│LED3_G  │ LED3 Green (预留)
  VCC  │3.3V
  GND  │GND
```

PMOD-A: 蓝牙CH9143 (4线: VCC+GND+TXD+RXD+BLESTA)
PMOD-B: 双RGB灯 (8 IO全用, 2灯独立 + 第3灯预留)

```
PMOD Pin │ Signal    │ Description
─────────┼───────────┼─────────────────
IO[0]    │ LED1_R    │ LED1 Red PWM
IO[1]    │ LED1_G    │ LED1 Green PWM
IO[2]    │ LED1_B    │ LED1 Blue PWM
IO[3]    │ LED2_R    │ LED2 Red PWM
IO[4]    │ LED2_G    │ LED2 Green PWM
IO[5]    │ LED2_B    │ LED2 Blue PWM
IO[6]    │ LED3_R    │ LED3 Red PWM (可选第3灯)
IO[7]    │ LED3_G    │ LED3 Green PWM (可选第3灯)

```

8 IO 100%利用：双RGB灯独立控制(2灯×3通道=6 IO) + 预留2 IO(LED3 RG)。
模式切换完全通过UART协议CMD字段控制，不占硬件IO。

## UART指令协议 (审查后修订: 加帧同步+校验+上行)

### 下行帧 (APP → FPGA)
```
FRAME: 0xAA | CMD | R | G | B | XOR
[0]    帧头(0xAA) — 帧同步
[1]    CMD[7:6]=模式(00=静态,01=呼吸,10=流水,11=保留) | CMD[5:0]=保留
[2]    R[5:0] — Red 6bit色深
[3]    G[5:0] — Green 6bit色深
[4]    B[5:0] — Blue 6bit色深
[5]    XOR = 0xAA ^ CMD ^ R ^ G ^ B — 校验

示例: 0xAA 0x00 0x3F 0x00 0x00 0x95 → 静态红色(满亮度, 校验通过)
      0xAA 0x40 0x3F 0x20 0x00 0x?? → 呼吸模式，橙色基准
```

### 上行帧 (FPGA → APP, 状态回传)
```
FRAME: 0xBB | STATUS | R_CUR | G_CUR | B_CUR | XOR
[0]    帧头(0xBB)
[1]    STATUS[7:6]=当前模式 | STATUS[5:0]=错误码(0=正常)
[2-4]  当前RGB值
[5]    XOR校验
```

帧头错位恢复: 接收状态机检测到非0xAA/0xBB帧头时，逐字节滑动直到重新锁帧。

## 模块接口规范 (审查新增)
- 时钟: `clk` (50MHz系统时钟)
- 复位: `rst_n` (低有效异步复位)
- 数据握手: `xxx_vld` (valid信号, 高有效)
- PWM参数: `parameter WIDTH = 8` (可配置位宽)
- 模块列表:
  - `uart_rx.v` — UART接收+帧同步
  - `uart_tx.v` — UART发送(状态回传)
  - `cmd_decoder.v` — 帧解析+校验+指令解码
  - `effect_engine.v` — 效果状态机(静态/呼吸/流水)
  - `pwm_generator.v` — PWM生成(参数化位宽)
  - `rgb_controller_top.v` — 顶层互联+跨时钟域同步

## 呼吸灯算法 (审查新增)
PWL (Piecewise Linear) 逼近sin曲线:
- 16-entry ROM: 每entry存一段斜率(4bit)
- 周期: 约3秒完整呼吸周期(PWM周期累加器控制)
- 输出: 8bit占空比 = base_color × sin_table[phase]
- 粉色基准: R=0xFF, G=0x60, B=0x80 → 暖粉呼吸

## PWM参数 (审查新增)
- 分辨率: 8bit (256级)
- 刷新率: 200Hz (无可见闪烁)
- 计数器时钟: 51.2kHz (50MHz / 976分频)
- 实现: 参数化设计, `pwm_generator #(.WIDTH(8))`

## 时钟架构明细
- 系统时钟: 50MHz (Cyclone IV EP4CE15F17C8 板载晶振)
- 波特率发生器: 50MHz / (16 × 115200) ≈ 27.1 → 取 27 分频 → 实际 115.74k bps (误差 0.47%, 可接受)
- 呼吸灯 PWL 时间基准: 50MHz / 2^18 ≈ 190Hz 累加器 → 3秒完整呼吸周期
- PWM 计数时钟: 50MHz / 976 ≈ 51.2kHz (8bit@200Hz刷新率)

## 蓝牙模块: CH9143 BLE
- 芯片: 南京沁恒 CH9143 BLE/UART/USB 三通
- 协议: BLE 4.2, GATT 透传
- 默认波特率: 115200 bps, 8N1
- AT 引脚: 低电平=AT命令模式, 高电平=透传模式 (正常工作拉高)
- BLESTA 引脚: 输出连接状态, 高=已连接 → 接FPGA IO做硬件断连检测
- 手机端: WCH 官方 BleUart SDK (虚拟串口, 与经典蓝牙SPP代码兼容)
- FPGA 接线: VCC(3.3V) + GND + TXD + RXD + BLESTA = 5线

## LED 驱动电路 (Outside Voice 新增)
- PMOD IO 输出: 3.3V, ~20mA max per pin
- RGB LED: 共阴, 每通道串 150Ω 限流电阻 → IF≈12mA
- 高亮 LED 备选: 加 NPN 三极管(2N3904)驱动, PMOD IO → 基极电阻 1kΩ
- PCB 上预留给限流电阻和驱动管位置
