# RGB 灯控 APP — UI 设计规范

由 /design-review 转 UI 设计规范 (无活页面的项目)
生成日期: 2026-06-01
状态: DRAFT — 等 APP 可运行后跑完整 /design-review

---

## 设计目标

课堂演示场景：教授和同学看到你拿出手机、连上蓝牙、滑动色轮、灯秒变。
APP 需要在 3 秒内传递"这是一个正经产品"的感觉，而非"学生作业糊的界面"。

核心原则：
- **色轮是主角**：打开 APP 第一个看到的就是色轮，不需要任何说明就能上手
- **暗色为主**：灯光控制 APP 用暗色背景，让颜色在屏幕上更鲜艳，也省电
- **少即是多**：4 个页面，每个页面一个核心任务

## 色彩系统

```
深色背景 (Dark Theme):
  --bg-primary:    #0F0F14      (最深背景，色轮页面)
  --bg-secondary:  #1A1A23      (卡片/面板背景)
  --bg-tertiary:   #252532      (输入框/选中态)
  --text-primary:  #F0F0F5      (主文字)
  --text-secondary:#8E8E9A      (辅助文字)
  --text-disabled: #4E4E5A      (禁用文字)
  --accent:        #6C5CE7      (主强调色，紫色——灯控类APP经典选择)
  --accent-warm:   #FF6B6B      (暖色强调，错误/删除)
  --success:       #00D68F      (成功/已连接)
  --divider:       #2A2A38      (分割线)

浅色背景 (Light Theme — 备选):
  --bg-primary:    #F8F8FC
  --bg-secondary:  #FFFFFF
  --text-primary:  #1A1A2E
  --text-secondary:#6E6E7E
  (accent/success/warning 与暗色主题共用)
```

**配色规则**：
- 整个 APP 不超过 12 种非灰色调——目前 4 种(紫/红/绿 + 保留一个警告黄)
- 主色调只用在交互元素上(按钮/选中态/滑块)，不做大面积纯色背景
- 禁止蓝紫渐变(这是 AI Slop 检测项第 1 条)

## 字体

APP 使用系统默认字体(不引入自定义字体文件，保持 APK 小)：

| 平台 | 字体 |
|------|------|
| Android | Roboto (默认) → 但要在标题上用 **sans-serif-medium** 变体 |
| iOS | SF Pro Display (默认) → 标题用 **Semibold** weight |

备选：如果追求差异化，引入 **Inter** (Variable) —— 一个 300KB 的 woff2 覆盖所有字重。但不是必须的。

**字体层级**：
```
页面标题:   24sp / 700 (Bold)      — 每页一个
区块标题:   18sp / 600 (Semibold)  — 如"当前模式""已保存场景"
正文:       16sp / 400 (Regular)   — 主要内容
辅助文字:   14sp / 400             — 标签/说明
小字:       12sp / 400             — 状态栏/蓝牙地址
```

## 四个页面设计

### 页面 1: 色轮控制页 (主页)

```
┌─────────────────────────────────┐
│  ┌─────────────────────────┐    │  顶部状态栏
│  │  RGB 灯控    [蓝牙图标]  │    │  蓝牙连接状态(已连接=绿色圆点)
│  └─────────────────────────┘    │
│                                 │
│       ┌─────────────┐           │
│       │             │           │
│       │   色轮      │           │  直径 ≥ 240dp
│       │   (HSV)     │           │  中心显示当前选中颜色预览
│       │             │           │
│       └─────────────┘           │
│                                 │
│  ┌─ 当前颜色 ─────────────────┐  │
│  │  [████████]  #FF4488       │  │  颜色预览条 + HEX值
│  └────────────────────────────┘  │
│                                 │
│  ┌─ 亮度 ─────────────────────┐  │
│  │  ○────────────●──────────  │  │  亮度滑块(0-100%)
│  └────────────────────────────┘  │
│                                 │
│  ┌──────────┐ ┌──────────────┐ │
│  │ 静态色    │ │ 呼吸         │ │  模式切换(当前选中高亮)
│  └──────────┘ └──────────────┘ │
│  ┌──────────┐ ┌──────────────┐ │
│  │ 流水      │ │ (预留)       │ │
│  └──────────┘ └──────────────┘ │
│                                 │
│         [  发送颜色  ]          │  主按钮，accent色
│                                 │
└─────────────────────────────────┘
```

**交互细节**：
- 色轮拖拽时，底部颜色预览条实时更新 HEX 值
- "发送颜色"按钮按下时：按钮文字变"已发送 ✓" → 0.5s 后恢复。短震动反馈(haptic)
- 模式切换时：当前模式卡片边框变 accent 色，其他保持透明
- 亮度滑块：实时发送(debounce 100ms)，不用再按发送按钮

**禁止事项**：
- 不用 emoji 做设计元素
- 模式切换卡片不用彩色圆圈图标
- 色轮不要带气泡/装饰性 SVG

## 交互规范

| 场景 | 行为 |
|------|------|
| 按钮按下 | 缩放 97% + 颜色加深 → 松开回弹。动画 100ms ease-out |
| 页面切换 | 右滑进入(下一页) / 左滑退出(返回)。动画 250ms |
| Toast | 底部浮现，2秒自动消失。成功=绿色, 失败=红色, 中性=灰色 |
| 蓝牙断连 | 顶部红色横幅"连接已断开，尝试重连..." + 自动重试3次 |
| 发送指令失败 | Toast "发送失败，请检查蓝牙连接" + 不改变UI状态 |
| 色轮拖拽 | 无延迟跟随手指。抬起时触发 debounce 100ms 后发送 |

## 触控目标

- 所有可交互元素最小 48dp (Android) / 44pt (iOS)
- 色轮为整个圆形区域可交互
- 亮度滑块轨道高度 ≥ 12dp

---

## 不做的

- 不使用紫色/蓝色渐变背景 (AI Slop #1)
- 不使用图标+彩色圆圈的 3 列网格布局 (AI Slop #2)
- 不使用 emoji 作为设计元素 (AI Slop #7)
- 不写"欢迎使用 RGB 灯控"之类的废话文案 (Happy Talk)
- 不使用 system-ui 或 -apple-system 作为主要展示字体——用平台原生 Roboto/SF Pro
